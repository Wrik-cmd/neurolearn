Binary Search

Hard ⏱ 12 min read 📊 O(log n) complexity

Introduction

Binary search is a fundamental algorithm that efficiently finds a target value within a sorted array. Instead of checking every element linearly, it repeatedly halves the search space — achieving a remarkable O(log n) time complexity.

💡
Key insight: Binary search only works on sorted data. Always verify your input is sorted before applying this algorithm.

Core Concept

The algorithm works by maintaining two pointers — left and right — that define the current search range. At each step, it examines the middle element:

• If the middle equals target → found! Return index.
• If middle < target → search right half.
• If middle > target → search left half.

Code Examples

JavaScript 
function binarySearch(arr, target) {
  let left = 0;
  let right = arr.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);

    if (arr[mid] === target) {
      return mid;          // Found! Return index
    } else if (arr[mid] < target) {
      left = mid + 1;      // Search right half
    } else {
      right = mid - 1;     // Search left half
    }
  }

  return -1;               // Not found
}
✅
Avoid overflow: Use mid = left + Math.floor((right - left) / 2) instead of (left + right) / 2 in languages with integer overflow.

Best Practices

Template Variations

Binary search has several template variants depending on whether you need to find the exact element, leftmost occurrence, or rightmost occurrence. Mastering each template is key to solving complex problems.

⚠️
Off-by-one errors are the #1 source of bugs in binary search. Carefully consider whether to use <= or < in your while condition, and whether to update mid ± 1.

Summary

Binary search is one of the most important algorithms to master. Its O(log n) complexity makes it dramatically faster than linear search for large datasets. Practice it until it's automatic — it appears frequently in coding interviews.
